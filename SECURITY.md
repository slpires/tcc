# Security Policy

## Escopo

Este projeto é de uso restrito e acadêmico. Ainda assim, práticas básicas de segurança devem ser respeitadas.

## Reporte de Vulnerabilidades

- Utilize o e-mail: slpires@slpires.com
- Não abra issues públicas contendo dados sensíveis.

## Boas Práticas

- Nunca versionar `.env`, `config.php` ou credenciais.
- Evitar armazenar dados sensíveis em logs.
- Manter consistência entre DEV e PRD sem expor informações privadas.

## Política de Correção

Problemas de segurança possuem prioridade. Correções devem ser documentadas no CHANGELOG.md.
